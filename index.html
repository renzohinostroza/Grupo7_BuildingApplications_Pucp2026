📋 Descripción

Descripción General del Proyecto

Este proyecto implementa una solución de negocio automatizada para una planta cervecera industrial, permitiendo registrar, monitorear y analizar el cumplimiento del Plan Maestro de Producción (PPA), integrando líneas de producción, marcas, turnos, supervisores y variables de afectación operativa en una única plataforma digital centralizada.

La arquitectura propuesta está basada en la nube y diseñada para consolidar información operativa en tiempo real, optimizar su procesamiento mediante flujos ETL y proporcionar acceso seguro y flexible a través de aplicaciones low-code y herramientas modernas de inteligencia de negocios.

La solución integra tecnologías cloud que permiten a la planta cervecera contar con una plataforma integral, escalable y orientada a la toma de decisiones basada en datos:

🗄️ Azure SQL Database
🏭 Microsoft Fabric Data Factory & Warehouse
⚡ Cursor
📊 Microsoft Power BI

🗄️ Base de Datos

Data Layer

La base de datos del sistema PPA ha sido diseñada para gestionar de forma eficiente la información relacionada al cumplimiento del Plan de Producción, integrando datos de planificación, ejecución real y variables de afectación operativa.

Su estructura relacional y normalizada garantiza la integridad, consistencia y trazabilidad de los datos, permitiendo analizar el desempeño productivo y facilitar la toma de decisiones en un entorno analítico. Este modelo puede implementarse en entornos SQL, permitiendo almacenamiento seguro y acceso confiable desde herramientas como Power BI, Python o plataformas de analítica avanzada.

Tablas Principales del Modelo

📋

tb_plan_de_produccion

Registra la planificación operativa: fecha, volumen planificado en HL, marca, turno y supervisor responsable. Representa el baseline operativo sobre el cual se mide el cumplimiento.

⚙️

tb_produccion_real

Contiene los resultados reales: volumen producido, producción programada e indicador de cumplimiento diario (PPA). Permite evaluar el desempeño frente al plan establecido.

⚠️

tb_variables_presentadas

Registra las pérdidas de producción en hectolitros no producidos, asociadas a eventos o incidencias. Permite cuantificar el impacto real de las desviaciones.

🧩

tb_variables_de_afectacion

Catálogo de causas que generan pérdidas: paradas operativas, fallas mecánicas o problemas de suministro. Permite clasificar y analizar las causas raíz del incumplimiento del PPA.

🏭

tb_marcas

Contiene información de los productos fabricados, incluyendo nombre de marca y factores de conversión asociados a la producción (factor HL por botella).

🏗️

tb_lineas

Define las diferentes líneas operativas dentro de la planta, permitiendo segmentar la producción y analizar el desempeño por línea.

⏱️

tb_turnos

Registra la organización temporal de la operación: nombre del turno y horarios de inicio y fin. Permite analizar la producción según franjas operativas.

👷

tb_supervisores

Contiene la información de los responsables de la operación productiva, permitiendo asignar y analizar el desempeño por supervisor.

💡

Este modelo relacional permite integrar la planificación, la ejecución real y las variables de afectación en una única estructura, proporcionando una base sólida para el análisis descriptivo y la construcción de modelos predictivos enfocados en el cumplimiento del PPA.

Application Layer

Diagrama de Flujo

A continuación se muestra el diagrama de flujo que representa el funcionamiento de la aplicación, enfocado para el uso en la empresa.

1

Pantalla de Inicio

Se cuenta con una pantalla principal que da la bienvenida al usuario a la aplicación. En el menú se visualizan las 3 funcionalidades: Datos maestros, Plan de producción y Producción real.

2

Datos Maestros

Se muestran las tablas maestras, las cuales se pueden modificar. Al ingresar a cualquier tabla maestra se muestra una ventana donde se pueden editar los datos.

3

Plan de Producción

Al ingresar a la opción de plan de producción se filtra por semana y se visualiza desde el más reciente hasta el más antiguo. En este apartado se ingresa y programa la producción por supervisor y marca.

4

Producción Real

Pulsando la opción de Producción Real, se busca de acuerdo a fecha y supervisor. Se muestra el plan de producción asociado y se solicita ingresar la cantidad de HL producidos realmente.

💡 Al guardar los HL, aparece una ventana que pide confirmar si se desea agregar variables de afectación. Al presionar Sí, se despliega la ventana de variables; al presionar No, se redirige al menú principal.

🔄 Transformación

ETL Layer

El sistema emplea un dataflow automatizado de gestión de datos que integra diversas capas tecnológicas de Microsoft. Este flujo garantiza la adecuada integración, transformación y disponibilidad de la información para apoyar la toma de decisiones dentro de la organización.

Origen de Datos

La información es ingresada mediante la aplicación, donde los usuarios registran operaciones relacionadas con la producción planeada y real del día. Posteriormente, estos datos se almacenan en Azure SQL Database.

Pipeline de Transformación (Dataflow)

El proceso ETL (extracción, transformación y carga) se ejecuta mediante un Dataflow en Microsoft Fabric, encargado de depurar, estandarizar y enriquecer la información antes de almacenarla en el Data Warehouse.

Automatización con Notebooks

Después de finalizar la etapa de transformación, se ejecutan Notebooks en Python con el fin de que las notificaciones diarias sean enviadas a los supervisores al inicio del día para el correspondiente análisis.

📧 Envío de correo 1 — PPA Diario

El primer envío consiste en el cálculo del PPA diario al corte del día anterior. El cálculo se realiza mediante consulta SQL a las tablas del modelo, generando un correo automático como se visualiza a continuación:

📧 Envío de correo 2 — Reporte de Supervisores

El segundo envío consiste en la generación de un reporte con fecha al corte del día anterior que contiene los nombres de los supervisores y sus turnos asociados. La finalidad es comparar los Hectolitros planificados con la producción real.

🏛️ Almacenamiento

DWH Layer

En el proyecto, esta capa recibe los datos procesados de la capa ETL y los almacena en Azure SQL Database. Los datos son normalizados para realizar consultas y visualizaciones.

Dentro del Data Warehouse, además de las tablas ya explicadas, se generan las relaciones entre las mismas, la creación de una tabla calendario, medidas y una tabla de reporte que sirve para realizar el Business Case del proyecto. Todo esto permite un correcto control de filtros y visualizaciones de la información en tiempo real.
